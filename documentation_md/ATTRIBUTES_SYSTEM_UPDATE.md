# Система атрибутов товаров - Обновление

## Описание изменений

Переделана система управления и отображения атрибутов товаров в админке и на витрине.

## 1. Админка (ProductsPage.tsx) - Вкладка АТРИБУТЫ

### Новая функциональность:
- **Группировка по категориям**: Атрибуты автоматически фильтруются в зависимости от выбранной категории товара
- **Чекбоксы для булевых атрибутов**: Атрибуты без `suffix` отображаются как простые чекбоксы (Да/Нет)
- **Чекбокс + Input для атрибутов со значениями**: Атрибуты с `suffix` (мм, кг, шт и т.д.) имеют чекбокс + поле ввода значения
- **Сохранение в formData.attributes**: Все выбранные атрибуты сохраняются в объект `formData.attributes`

### Как это работает:
1. При выборе категории в админке отображаются только релевантные атрибуты
2. Для булевых атрибутов (например, "Крышка", "Замок двери"):
   - Чекбокс включает/выключает атрибут
   - При включении сохраняется значение `"true"`
3. Для атрибутов со значениями (например, "Толщина стали 2 мм"):
   - Чекбокс включает/выключает атрибут
   - При включении появляется input для ввода значения
   - Суффикс отображается справа от input (мм, кг, шт и т.д.)

### Пример использования:
```
Категория: Мангалы
- ☑ Толщина стали [2] мм
- ☑ Дымоход [800] мм
- ☐ Крышка
- ☑ Решётка-гриль
```

## 2. Страница товара (Product.tsx) - Раздел "Атрибуты товара"

### Новая функциональность:
- **2-столбцовая сетка**: Атрибуты выводятся в 2 столбца (grid-cols-2)
- **Иконки из attributeMeta**: Каждый атрибут отображается с соответствующей иконкой
- **Полная информация**: Название атрибута + значение с суффиксом

### Структура отображения:
```
[Иконка] Название атрибута
        Значение + суффикс
```

### Пример:
```
[⚙] Толщина стали          [🔧] Дымоход
    2 мм                       800 мм

[📦] Решётка-гриль         [🔥] Дровница
    Да                         Да
```

## 3. Карточка каталога (Catalog.tsx)

### Без изменений:
- Остаются 2 featured_attributes как было
- Отображаются в компактном виде на карточке товара

## Технические детали

### Используемые данные:
- `attributesByCategory` - группировка атрибутов по категориям
- `attributeMeta` - метаинформация об атрибутах (название, иконка, суффикс)

### Структура attributeMeta:
```typescript
{
  thickness_steel: { 
    label: "Толщина стали", 
    icon: Layers, 
    suffix: " мм" 
  },
  kazan_ring: { 
    label: "Кольцо под казан", 
    icon: CircleDot 
    // без suffix - булев атрибут
  },
  // ...
}
```

### Структура attributesByCategory:
```typescript
{
  grill: ["thickness_steel", "chimney_height", "skewers_capacity", ...],
  dog_cage: ["bars_profile", "door_lock", "tray", ...],
  // ...
}
```

## Сохранение данных

Атрибуты сохраняются в поле `attributes` товара:
```json
{
  "id": "mangan_classic_800",
  "name": "Мангал Classic 800",
  "category": "grill",
  "attributes": {
    "thickness_steel": "2",
    "chimney_height": "800",
    "kazan_ring": "true",
    "lid": "true"
  }
}
```

## Миграция существующих данных

Если у вас уже есть товары с атрибутами в старом формате, они будут работать как есть. Новая система совместима с существующими данными.

## Тестирование

1. **Админка**: 
   - Откройте товар в админке
   - Перейдите на вкладку АТРИБУТЫ
   - Выберите категорию
   - Проверьте, что отображаются правильные атрибуты
   - Включите несколько атрибутов и сохраните

2. **Витрина**:
   - Откройте товар на витрине
   - Проверьте раздел "Атрибуты товара"
   - Убедитесь, что атрибуты отображаются в 2 столбца с иконками

3. **Каталог**:
   - Проверьте, что featured_attributes отображаются на карточках товаров
